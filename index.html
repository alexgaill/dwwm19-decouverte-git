Commandes git

Git init

On initialise le dépôt (repository) grâce à la commande git init

git config

git config permet de définir la configuration de git sur votre machine. Cette commande n'est à utiliser que lorsque vous souhaitez la modifier et non à chaque projet

On peut par exemple la première fois faire git config --global (on modifie la configuration au global sur notre machine) user.name "mon nom"

git config --global user.email "mon@email.fr" pour changer l'email de l'auteur

git add

Permet d'ajouter les nouveaux fichiers (untracked) et les fichiers ayant des changements (modified) à la liste des fichiers à ajouter à la nouvelle version

La commande utilisée est git add [nom_du_fichier]

Une fois utilisée, les fichiers deviennent added

git add . est la commande générique qui permet d'ajouter tous les fichiers

git commit

Une fois les fichiers ajoutés on utilise git commit pour valider la version à laquelle on ajoute une description

La commande est git commit -m "Description"

git log

git log permet d'obtenir les log des commit donc l'historique de notre projet

Chaque commit a différentes informations: les infos de l'auteur, un id unique et pour le commit le plus récent => (HEAD-> master) qui indique que sur notre branche master la dernière version visible est celle-ci

git branch

Permet d'afficher et de gérer les branch

git branch affiche les branches

git branch -M modifie le nom d'une branch [nouveau_nom_de_branch]

git branch -C ajoute une branch [nom_de_la_nouvelle_branch]

git branch -D supprime une branch [nom_de_la_branch_a_supprimer]

git checkout/ git switch

Permet de naviguer entre nos branch

La commande est git checkout [nom_de_la_branch_a_atteindre]

git merge

Une fois qu'un travail est fini sur une branch, on veut associer le résultat à une autre branch (ex: master)

On se rend sur la branch qui va recevoir les infos puis on utilise git merge [nom_de_la_branch_a_merger]

La branch qui reçoit s'actualise et récupère les nouvelles infos

git reset

Permet de revenir à un ancien commit pour reprendre le travail sur une base plus saine

git reset [id_du_commit]

git stash

Permet de mettre en mémoire tampon le travail effectué et non commit pour effectuer d'autres opérations comme retourner sur une autre branch

Permet également de déplacer ce travail sur une autre branch

git stash pop ressort toutes les données mises en mémoire
